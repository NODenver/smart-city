前端AI竞赛题目 - 智慧城市运行态势感知系统
一、竞赛背景
随着新型智慧城市建设加速推进，城市治理正从“被动响应”向“主动预警、智能决策”转型。城市管理涉及海量多源异构数据（如市民上报、物联设备、视频监控等），亟需通过 AI 赋能实现态势感知、问题识别与辅助决策。
本次竞赛聚焦前端+AI融合能力，考察参赛者在真实业务场景下，综合运用现代前端技术栈、大模型 API 与数据可视化手段，构建一个面向城市管理人员的运行态势感知与智能辅助决策平台。


二、竞赛主题
智慧城市运行态势感知系统
目标：构建一个集 多源数据接入、实时态势展示、AI智能分析、决策建议生成 于一体的前端系统。
重要说明：
	本竞赛提供两份 JSON 格式 Mock 数据（非 Excel），模拟真实城市数据源；
	参赛者无需对接真实后端或物联网平台，确保竞赛公平性与可实施性；
	所有数据均为脱敏模拟数据，不涉及真实隐私或敏感信息。

四、功能需求（总分100分）
4.1 核心功能（必做，70分）
4.1.1 多源数据导入与结构化展示（15分）
	提供两份 JSON Mock 数据文件（见第六部分）：
¢	city_events.json：市民上报的城市问题（如井盖缺失、占道经营）
¢	sensor_data.json：物联网设备采集的环境/设施状态（如积水深度、路灯故障）
	支持 拖拽上传 或 点击上传 JSON 文件
	自动解析并校验数据格式，错误时友好提示
	展示两类数据的统计概览：
¢	事件总数、类型分布、区域分布
¢	传感器异常数量、设备在线率
	支持切换查看：仅事件 / 仅传感器 / 合并视图
	列表支持分页或虚拟滚动（数据量 > 500 条）
4.1.2 AI智能分析与建议生成（20分）
	问题聚类与热点识别：
¢	自动识别高频问题类型（如“道路积水”、“噪音扰民”）
¢	结合地理区域，发现“问题高发区”
	异常关联分析：
¢	将传感器异常（如积水>30cm）与附近市民上报事件自动关联
¢	标记“高置信度关联事件”
	AI辅助决策（核心）：
¢	用户可选择一条或多条事件/异常，点击“AI分析”
¢	调用大模型 API，输入结构化数据（含位置、类型、时间、传感器值等）
¢	输出内容包括：
	问题归因（如“该区域排水管网老化导致积水”）
	处置建议（如“建议调度排水车，并安排管网检修”）
	优先级评估（高/中/低）
¢	支持查看历史 AI 分析记录
4.1.3 城市运行态势大屏（20分）
	设计 1920x1080 分辨率 的态势感知大屏
	包含以下至少 5 种可视化组件：
a.	城市问题热力图（基于区域坐标或行政区划）
b.	问题类型环形图（事件 vs 传感器异常分类对比）
c.	实时事件滚动列表（带“NEW”标识，模拟每10秒新增）
d.	关键指标卡片：今日事件数、异常设备数、高优先级问题数、平均响应时长（模拟）
e.	趋势折线图：近7天问题数量变化
	支持深色科技感主题
	布局建议：顶部指标 + 左侧地图/热力 + 右侧图表 + 底部事件流
4.1.4 智能预警与处置跟踪（15分）
	预警规则配置（前端模拟）：
¢	同一区域1小时内上报≥5次同类事件 → 触发“聚集性问题”预警
¢	传感器数值连续3次超阈值 → 触发“设施异常”预警
	预警以 Toast 或侧边栏通知形式弹出
	预警列表支持标记“已处理”、“待跟进”
	每条预警可关联 AI 生成的处置建议


4.2 进阶功能（选做，30分）
4.2.1 AI对话助手（15分）
	集成聊天窗口，支持自然语言查询：
¢	“今天哪些区域积水最严重？”
¢	“列出所有高优先级的路灯故障”
	AI 助手调用大模型，结合当前加载的数据回答
	支持上下文记忆（如追问“怎么处理？”）
4.2.2 自动生成处置日报（10分）
	AI 自动生成当日《城市运行简报》
	内容包含：问题汇总、热点区域、处置建议、预警回顾
	支持导出为 PDF（使用 jsPDF/html2canvas）
4.2.3 多维筛选与对比（5分）
	按时间、区域、问题类型、数据源（事件/传感器）组合筛选
	支持对比两个时间段的问题变化
